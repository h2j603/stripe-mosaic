# 혁의 디자인 시스템 원칙

## 배경 & 색상

- 배경은 **회색 계열** 기본. 검정(`#000`, `#0a0a0a`)이나 남색 계열 금지.
  - 다크: `#1a1a1a` ~ `#2a2a2a` 범위 권장
  - 라이트: `#f4f4f4` ~ `#fafafa` 범위 권장
- **포인트 컬러는 반드시 먼저 상의 후 사용.** 새 프로젝트 시작 시 "어떤 컬러 팔레트로 할까요?" 묻기.
- 남색·네이비·보라 계열 배경 금지. 바이브코딩 느낌 나는 색상 지양.

## 레이아웃 & 구조

- 공간 구분은 **라인(`border-bottom` 1px)과 타이포그래피 위계**로.
  → 카드 박스, 컨테이너 div를 무분별하게 만들지 않는다.
- 섹션 구분에 배경색 깔린 컨테이너 지양. `border-top` 또는 여백으로 대신한다.
- 그리드는 항상 존재해야 한다. 임의 배치 금지.

## 금지 효과

- **글래스모피즘** 금지 (`backdrop-filter: blur`, 반투명 배경 레이어)
- **그림자** 금지 (`box-shadow`, `drop-shadow`, `text-shadow`)
- **글로우/네온** 효과 금지 (`filter: blur`, 색상 glow)
- 장식용 그라디언트 금지. 기능적 용도 외 사용 안 함.

## 위계 표현 방법

- 위계는 **font-size + font-weight 조합**으로만 표현
- 색상·명도 차이로 위계 만들지 않는다 (밝은 회색 = 덜 중요, 어두운 회색 = 더 중요 → 금지)
- 활성/비활성 상태: 색 변경 아닌 font-weight 차이로
- 분류 구분(카테고리, 태그): 색상 pill 허용 — 이건 위계가 아닌 분류

## 타이포그래피

- 위계는 **크기 + 굵기 + 색상 명도**로만 구분. 효과 쓰지 않는다.
- 폰트: Pretendard 기본 (웹), Noto Sans KR 대체
- 자간: 본문 기본값. 레이블/캡션만 `letter-spacing: 0.05em` 허용.
- 행간: 본문 1.6, 헤딩 1.2

## 컴포넌트

- 버튼: 플랫. 외곽선 버튼(`border`) 또는 솔리드. 그림자 없음.
- 인풋: 하단 라인(`border-bottom`)만으로 표현하는 방식 권장.
- 구분선: `1px solid` + 회색 계열 (`#e0e0e0` 라이트 / `#2e2e2e` 다크)
